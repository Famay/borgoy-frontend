# Дорожная карта VerMeat

## Статус

Базовый MVP собран: в проекте есть React + TypeScript фронтенд, Express API, PostgreSQL/Prisma, загрузка сертификатов в Pinata/IPFS, фиксация хеша в Polygon Amoy, QR-проверка, админский журнал и Docker/Nginx контур для деплоя.

Документ фиксирует текущую базу, ближайшие задачи и дальнейшие цели после MVP.

## Цель продукта

Сделать рабочую систему цифровой верификации и идентификации боргойской баранины:

- поставщик создает карточку партии и загружает сертификат;
- сервер рассчитывает SHA-256 от содержимого файла;
- файл сертификата отправляется в Pinata/IPFS, в базе сохраняется CID;
- хеш сертификата фиксируется в смарт-контракте Polygon Amoy;
- система генерирует QR-код публичной проверки;
- покупатель проверяет сертификат по QR-токену, номеру партии или номеру сертификата;
- администратор контролирует статусы сертификатов, журнал действий и качество данных.

## Текущая структура

- `src/` - фронтенд на React + TypeScript.
- `server/` - Express API, JWT, bcrypt, Zod, Multer.
- `prisma/` - схема Prisma, миграции PostgreSQL, начальные данные.
- `contracts/` - Solidity-контракт, Hardhat-скрипты, тесты.
- `test/` - тесты смарт-контракта.
- `ignition/` - модуль деплоя Hardhat Ignition.
- `docker-compose.yml` - PostgreSQL, бекенд, фронтенд/Nginx.
- `nginx/` - производственный обратный прокси.

## Реализованные модули бекенда

- `AuthModule` - регистрация, вход, JWT-токен доступа, bcrypt-хеш пароля.
- `UsersModule` - поставщики, администраторы, статусы учетных записей.
- `BatchesModule` - партии продукции: номер, продукт, происхождение, дата, масса.
- `CertificatesModule` - метаданные сертификата, серверный SHA-256, статусы, проверка дублей.
- `FileStorageService` - загрузка в Pinata/IPFS и получение CID.
- `BlockchainService` - вызовы смарт-контракта через ethers.js в Polygon Amoy.
- `QRCodeService` - QR-код со ссылкой на публичную проверку.
- `PublicCheckModule` - публичная проверка без авторизации.
- `AdminModule` - сводка, журнал аудита, смена статуса и удаление сертификатов.
- `AuditLogModule` - журнал действий и ошибок.

## API-интерфейс

- `GET /api/health`
- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`
- `GET /api/batches`
- `POST /api/batches`
- `GET /api/certificates`
- `GET /api/certificates/check-file?hash=...`
- `GET /api/certificates/:id`
- `POST /api/batches/:batchId/certificates`
- `GET /api/public/verify/:token`
- `GET /api/public/verify?query=...`
- `GET /api/public/verify?batchNumber=...`
- `GET /api/public/verify?certificateNo=...`
- `GET /api/admin/overview`
- `GET /api/admin/audit-logs`
- `PATCH /api/admin/certificates/:certificateNo/status`
- `DELETE /api/admin/certificates/:certificateNo`

## Сущности базы данных

- `User` - пользователь, поставщик или администратор.
- `Batch` - партия продукции.
- `Certificate` - сертификат партии.
- `BlockchainTransaction` - запись о транзакции фиксации сертификата.
- `VerificationResult` - результат публичной проверки.
- `AuditLog` - журнал системных и пользовательских действий.

## Интерфейс смарт-контракта

- `registerCertificate(bytes32 certificateId, bytes32 documentHash, string cid)` - регистрация сертификата.
- `getCertificate(bytes32 certificateId)` - получение данных сертификата.
- `isCertificateValid(bytes32 certificateId, bytes32 documentHash)` - проверка соответствия сертификата и хеша.

## Локальные команды

- `npm run dev` - запустить Vite-фронтенд.
- `npm run dev:api` - запустить Express API на `SERVER_PORT`.
- `npm run build` - проверить TypeScript-фронтенд и собрать Vite-приложение.
- `npm run lint` - запустить ESLint.
- `npm run server:typecheck` - проверить бекенд-код TypeScript-компилятором.
- `npm run prisma:generate` - сгенерировать Prisma Client.
- `npm run prisma:migrate` - применить миграции к локальной PostgreSQL.
- `npm run prisma:deploy` - применить производственные миграции.
- `npx prisma db seed` - загрузить демо-поставщика, админа, партию и сертификат.

## Команды смарт-контракта

- `npm run hardhat:compile` - скомпилировать Solidity-контракт.
- `npm run hardhat:test` - запустить локальные тесты Hardhat.
- `npm run hardhat:deploy:amoy` - задеплоить `CertificateRegistry` в Polygon Amoy.
